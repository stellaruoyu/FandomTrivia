Group Card Pattern

The Holidays page isn't special — it follows a reusable blueprint that every "group" of quizzes shares.

Pattern Recognition

If you visited the site last week, you might have noticed the Disneyland page — a "selector" screen listing all Disney movie quizzes with fancy cards. This week's Holidays page uses the exact same pattern.

The design pattern is called a Group Card: a universe that contains multiple individual quizzes, each shown as a clickable card. The pattern has three layers: the universe group definition (in constants.ts), the selector component (in App.tsx), and the route connecting them.

Good To Know

This pattern is why you can tell AI "add a new universe group like the Holidays one" and it knows exactly what to do. The pattern is so consistent that adding a new 6-quiz universe is almost copy-paste work.

Inside the Selector Component

The HolidaysSelector component lives inside src/App.tsx alongside selectors for Disneyland, Anime, and other grouped universes. Each selector follows the same format:

CODE 

const HolidaysSelector = () => {
  const navigate = useNavigate();
  const holidays = [
    { label: 'Independence Day',
      title: '4th of July',
      desc: '10 questions on the Declaration & fireworks',
      icon: '🎆',
      view: 'trivia-july-4th',
      gradient: 'from-blue-600/20 to-red-900/20' },
    // ... 4 more holidays
  ];
PLAIN ENGLISH

The selector is a React component — a self-contained building block for the UI.

useNavigate is the hook that lets users click and jump to a different page.

Inside the component, we define a list of holiday quizzes...

...each with a display label, title, short description...

...an emoji icon, the route name it links to...

...and a color gradient for the card's background.

The array is just data — the rendering code loops over it to create the visual cards.

What About the Head?

Each selector page also includes a Helmet section that sets the page title, meta description, and SEO tags. This is critical for showing up in Google searches:

CODE 

<Helmet>
  <title>Holiday Quizzes | Fandom Trivia</title>
  <meta name="description"
    content="Test your holiday knowledge..." />
  <link rel="canonical"
    href="https://www.fandom-trivia.com/selector-holidays" />
</Helmet>
PLAIN ENGLISH

The Helmet component injects a title into the browser tab.

It also adds a meta description — the snippet Google shows in search results.

And a canonical link tells search engines this is the official URL for this page, preventing duplicate content penalties.
