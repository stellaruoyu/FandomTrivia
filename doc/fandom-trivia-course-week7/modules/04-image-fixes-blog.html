Image Fixes & Blog Posts

Broken images from Wikipedia were replaced with local copies, and new blog content was added for SEO.

The Image Problem

Many trivia cards used images from Wikimedia Commons — the free image repository. There's just one problem: Wikimedia URLs expire or change, and the site can't control them. When the images go down, the cards look broken.

This week, all Wikimedia-hosted images were replaced with local copies stored in the project's /public/images/ directory. This is a classic external dependency cleanup: bringing assets in-house so they never break unexpectedly.

🔌
The Rule of External Dependencies

Whenever you link to something you don't control (an image URL, an API, a CDN), you're accepting the risk that it goes away. The fix is always the same: own your assets. Copy them locally and serve them yourself.

Before & After: Image Paths

The change is simple but impactful. Instead of linking to Wikipedia, images now reference a local file:

BEFORE 

// Wikimedia URL (external — can break)
image: 'https://upload.wikimedia.org/
  wikipedia/en/3/3d/
  The_Lion_King_poster.jpg'
AFTER

// Local file (we control it)

image: '/images/holidays.jpg'

Instead of a long URL pointing to someone else's server, the path now points to a file in our own /public/images/ folder. Shipped with the app, always available.

Why Blog Posts Matter

Every blog post is a new page that search engines can index. "World Cup trivia" and "Christmas quiz" are high-volume search terms. By publishing content around these topics, the site attracts visitors who wouldn't have found it otherwise — a classic SEO growth play.

How a Blog Post Works

Blog posts live in src/blogPosts.ts as data objects, not separate HTML files. Each post has a slug (the URL), a title, a date, a short preview, and the full content:

CODE 

{
  slug: 'holiday-trivia-is-live',
  title: 'Holiday Trivia Is Live!',
  date: 'June 18, 2026',
  preview: 'Celebrate the seasons with 5
    new holiday trivia quizzes...',
  content: 'We are thrilled to announce
    the launch of our Holidays universe!'
}
PLAIN ENGLISH

Each blog post is a data object — a simple bundle of fields.

The slug determines the URL: /blog/holiday-trivia-is-live.

The title, date, and preview show up on the blog listing page.

The preview is what users see in the blog card before clicking...

...and the full content is the article body.
