Under the Hood: A Trivia File

Every quiz is a TypeScript file that exports an array of question objects. Here's the exact pattern from src/christmasTrivia.ts:

CODE 

import type { MCTriviaQuestion } from './constants';

export const CHRISTMAS_TRIVIA: MCTriviaQuestion[] = [
  {
    id: 1,
    question: "In which modern-day country was Saint Nicholas born?",
    options: ["Germany", "Norway", "Turkey", "Finland"],
    answer: "Turkey",
    evidence: "Saint Nicholas was born in Patara, a city in Lycia..."
  },
  // ... 9 more questions
];
PLAIN ENGLISH

First, we import the question type — a blueprint saying every question must have certain fields.

Then we create a list of questions called CHRISTMAS_TRIVIA.

Each question has a unique ID number...

...the question text (what the player sees)...

...four multiple-choice options...

...the correct answer...

...and a citation or evidence explaining the answer.

The bracket closes the list. Simple, right?

💡
Key Insight

Every trivia file follows exactly this pattern — question, options, answer, evidence. If you wanted to add a new quiz, you'd create a new file that looks almost identical to this one. The type system enforces it.

Hooking It Up: Registration

A trivia file doesn't automatically appear on the site. It has to be registered in src/constants.ts — the master configuration file that defines every category, universe, and quiz on the platform.

The Holidays universe is defined as a group entry alongside Disneyland, Anime, and others:

CODE 

{
  id: 'holidays',
  title: 'Holidays',
  tags: ['Festive', 'History', 'Celebration'],
  description: 'Celebrate the seasons! Test your knowledge...',
  image: '/images/holidays.jpg',
  buttonText: 'Celebrate Now',
  icon: 'Sparkles',
  path: '/selector-holidays'
}
PLAIN ENGLISH

Each universe entry is a config object with an internal ID, display title, and tags for filtering.

The description appears on the homepage card.

The image, button text, and icon control what the card looks like.

The path tells the app which page to navigate to when clicked — in this case, the Holidays selector page.

Bonus: Grading Tiers

Each holiday category also got its own grading system — funny themed ranks based on your score. For Christmas, you might be a "Grinch" (0-50%), a "Carol Singer" (50-70%), an "Elf on the Shelf" (70-90%), or "Santa Claus" (90%+). Each rank comes with a unique character description.

CODE 

const CHRISTMAS_GRADES = [
  { threshold: 90, label: 'Santa Claus',
    character: { name: 'Christmas Historian',
      image: '/images/christmas.jpg',
      desc: 'You are a true master of Christmas trivia!' }},
  { threshold: 70, label: 'Elf on the Shelf', ... },
  { threshold: 50, label: 'Carol Singer', ... },
  { threshold: 0, label: 'Grinch', ... }
];
PLAIN ENGLISH

Each grade has a minimum score (threshold) and a label.

90%+ earns you the top rank — Santa Claus.

70-89% is Elf on the Shelf — still excellent.

50-69% is Carol Singer — decent effort.

Below 50% gets the Grinch rank — time to study!
