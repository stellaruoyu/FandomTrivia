Week 2 — Quiz Content & UI Polish

February 22–23, 2026 · 6 commits · MC Quiz Components UI/UX Content

With the scaffold in place, Week 2 focused on what makes a trivia app worth using: great questions and polished UI. This week saw the creation of the MCQuizView component — the single most reused piece of UI in the app.

Key Concepts

Generic Multiple-Choice Quiz Component

The MCQuizView was designed as a generic, reusable component. Instead of hardcoding each quiz, the component accepts a questions array and handles all interaction logic.

interface MCQuizViewProps {
  questions: Question[];
  onComplete: (score: number, total: number) => void;
}

function MCQuizView({ questions, onComplete }: MCQuizViewProps) {
  const [current, setCurrent] = useState(0);
  const [score, setScore] = useState(0);
  const [answered, setAnswered] = useState(false);

  const handleAnswer = (index: number) => {
    if (answered) return;
    if (index === questions[current].correct) setScore(s => s + 1);
    setAnswered(true);
  };

  const next = () => {
    if (current >= questions.length - 1) {
      onComplete(score, questions.length);
    } else {
      setCurrent(c => c + 1);
      setAnswered(false);
    }
  };
  // ... render question, choices, feedback
}
Pattern: Generic MCQuizView receives data via props, manages internal state (current question, score, answered), and calls onComplete when done. This meant every new fandom could get a quiz by just passing different data.

Trivia Content Sourcing

Each quiz question was sourced with evidence. For example, the Twilight quiz cited specific book passages:

{
  question: "What is the name of the vampire clan that rules Volterra?",
  choices: ["The Volturi", "The Denali Coven", "The Amazon Coven", "The Egyptian Coven"],
  correct: 0,
  source: "Twilight, Chapter 15"
}

This source citation pattern became a standard across all universes. It adds credibility and lets curious players verify answers.

UI Polish: Smooth Scroll, Carousel, Leaderboard

Several UI features were added to improve the user experience:

  • Smooth scroll — scroll-behavior: smooth for anchor links
  • Branding — Consistent color palette, logo, typography
  • Carousel — Horizontal scrollable card list for universe selection
  • Leaderboard — Real score display from SQLite
  • Image fixes — Proper sizing and fallbacks for universe images

The carousel was a CSS-only implementation using overflow-x: auto and scroll-snap-type: x mandatory — no JS library needed.

.carousel {
  display: flex;
  overflow-x: auto;
  scroll-snap-type: x mandatory;
  gap: 1rem;
}
.carousel > * {
  scroll-snap-align: start;
  flex: 0 0 280px;
}

Reflection

This week taught the value of generic components. By making MCQuizView data-driven early, adding new trivia content became a matter of writing JSON rather than new UI. The UI polish (carousel, smooth scroll, leaderboard) transformed the app from "works" to "feels good." The source citation pattern added a layer of quality that sets this trivia app apart.
