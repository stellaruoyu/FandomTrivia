Week 4 — Content Expansion & Routing

February 25–28, 2026 · 4 commits · Routing Three-Body Leaderboard Game Logic

Week 4 was about breadth: more content, proper navigation, leaderboards, and core game logic. The Three-Body Problem series joined the roster, and the app started to feel like a real trivia platform with multiple screens and a scoreboard.

Key Concepts

React Router & Navigation

Navigation links were added for multiple pages: Home, Quiz, Leaderboard, Profile. React Router handled client-side routing:

import { BrowserRouter, Routes, Route } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <Navbar />
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/quiz/:universe" element={<QuizPage />} />
        <Route path="/leaderboard" element={<Leaderboard />} />
        <Route path="/profile" element={<Profile />} />
      </Routes>
    </BrowserRouter>
  );
}

The /quiz/:universe dynamic route allowed adding new quizzes without new route definitions — just new data in constants.

Universe Data Structure

Each fandom was modeled as a universe with metadata and questions:

interface Universe {
  id: string;
  name: string;
  description: string;
  image: string;
  color: string;
  questions: Question[];
}

const UNIVERSES: Universe[] = [
  {
    id: 'twilight',
    name: 'Twilight',
    description: 'Test your knowledge of Forks, Washington...',
    image: '/images/twilight.jpg',
    color: '#8B0000',
    questions: twilightQuestions,
  },
  // ... Harry Potter, K-Pop, Three-Body, Zootopia
];

This data-driven approach meant the Home page could render universe cards by mapping over UNIVERSES, and the Quiz page could look up the right questions by URL parameter.

Leaderboard Logic

The leaderboard read from Supabase, ordering by score descending:

const { data: scores } = await supabase
  .from('scores')
  .select('username, score, universe')
  .order('score', { ascending: false })
  .limit(20);

Key considerations:

  • Pagination — limit to top 20 to keep the UI clean
  • Real-time updates — after each quiz completion, refetch
  • Per-universe filtering — optional .eq('universe', id) filter

Three-Body Problem Content

The Three-Body Problem series (Liu Cixin) demanded a different kind of question — hard sci-fi with physics concepts:

{
  question: "What is the 'Sophon'?",
  choices: [
    "A proton unfolded into multiple dimensions",
    "A type of spaceship engine",
    "An alien communication device",
    "A biological weapon"
  ],
  correct: 0,
  source: "Three-Body Problem, Ch. 22"
}

This was also the week an image asset (threebody.jpg) was committed, adding visual identity to the universe.

Game Logic: Score Tracking

The quiz game logic started taking shape:

  • Question selection — random subset from the universe's question pool
  • Scoring — correct answer = 1 point, no negative marking
  • Progress tracking — current question index, answered state
  • Completion — show final score, save to Supabase, offer "play again"

Reflection

Week 4 was when the app structure stabilized. The UNIVERSES data array became the single source of truth for all trivia content. Adding a new universe was now just: 1) write questions, 2) add an entry to UNIVERSES, 3) add an image. No routing changes, no new components. This data-driven architecture paid dividends in later weeks when more content was added.
