Key Concepts

Badge System

Badges are achievements awarded for specific accomplishments. The evaluation logic checks conditions after each quiz completion:

interface Badge {
  id: string;
  name: string;
  description: string;
  icon: string;
  condition: (stats: UserStats) => boolean;
}

const BADGES: Badge[] = [
  {
    id: 'first_quiz',
    name: 'First Steps',
    description: 'Complete your first quiz',
    icon: '🌟',
    condition: (s) => s.totalQuizzes >= 1,
  },
  {
    id: 'perfect_score',
    name: 'Flawless Victory',
    description: 'Get 100% on any quiz',
    icon: '💎',
    condition: (s) => s.lastQuizPerfect,
  },
  {
    id: 'all_universes',
    name: 'Explorer',
    description: 'Play a quiz in every universe',
    icon: '🌍',
    condition: (s) => s.universesPlayed.length >= 5,
  },
  {
    id: 'century',
    name: 'Century Club',
    description: 'Answer 100 questions total',
    icon: '🏆',
    condition: (s) => s.totalQuestions >= 100,
  },
];

Badges are evaluated client-side after each game, then persisted to the user's profile in Supabase. The badge display in the dashboard shows earned vs locked badges with a progress grid.

Pattern: Each badge has a pure condition function that takes stats and returns boolean. This makes adding new badges trivial — just write the condition function.

Quiz History & Dashboard

Every quiz attempt was saved to Supabase and displayed in a history view:

interface QuizAttempt {
  id: string;
  user_id: string;
  universe: string;
  score: number;
  total: number;
  timestamp: Date;
}

// Fetch history
const { data: history } = await supabase
  .from('quiz_attempts')
  .select('*')
  .eq('user_id', user.id)
  .order('timestamp', { ascending: false })
  .limit(50);

The dashboard aggregated stats: total quizzes, average score, badges earned, favorite universe, and recent activity.

Username Modal & Profile

After signing in with Google, new users were prompted to set a username via a modal:

function UsernameModal({ userId, onComplete }) {
  const [username, setUsername] = useState('');
  const [error, setError] = useState('');

  const handleSubmit = async () => {
    if (username.length < 3) {
      setError('Username must be at least 3 characters');
      return;
    }
    const { error: dbError } = await supabase
      .from('profiles')
      .update({ username })
      .eq('id', userId);

    if (!dbError) onComplete(username);
  };
  // ... render modal overlay with input + validation
}

Validation ensured: 3+ characters, no special characters, unique username (checked against existing profiles).

Google Translate Integration

The app added Google Translate support for internationalization:

// Dynamically load the Google Translate script
const script = document.createElement('script');
script.src = 'https://translate.google.com/translate_a/element.js';
script.onload = () => {
  new google.translate.TranslateElement({
    pageLanguage: 'en',
    includedLanguages: 'es,fr,de,ja,ko,zh-CN',
  }, 'google_translate_element');
};
document.body.appendChild(script);

This was later removed to simplify the codebase, but it showed the app's international ambition.
