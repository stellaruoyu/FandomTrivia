Week 1 — Project Bootstrap

February 21, 2026 · 4 commits · Vite React Tailwind Vercel SQLite

This week was about going from a blank directory to a running web application. The stack choice was deliberate: Vite + React + TypeScript for a fast, type-safe frontend, Tailwind CSS for utility-first styling, and Express + better-sqlite3 for a local backend during development. Vercel was chosen for deployment.

Key Concepts

Vite + React + TypeScript Scaffolding

Vite is a build tool that offers near-instant HMR (Hot Module Replacement) during development and optimized production builds. The project was scaffolded with TypeScript for type safety.

npm create vite@latest . -- --template react-ts

Key files created:

  • vite.config.ts — Vite configuration with React plugin
  • tsconfig.json — TypeScript compiler options
  • index.html — Entry HTML file
Recipe: When scaffolding any new React project, always use --template react-ts to get TypeScript out of the box. Add Tailwind immediately via npm install -D tailwindcss @tailwindcss/vite.

Express + SQLite Local Backend

Before migrating to Supabase, the app used a custom Express server with SQLite for persistence. better-sqlite3 was chosen over other SQLite libraries for its synchronous API (simpler for a small project).

import Database from 'better-sqlite3';
import express from 'express';

const app = express();
const db = new Database('fandom.db');

app.get('/api/leaderboard', (req, res) => {
  const scores = db.prepare(
    'SELECT username, score FROM scores ORDER BY score DESC LIMIT 10'
  ).all();
  res.json(scores);
});

The API followed a simple REST pattern: Express routes backed by SQLite queries, returning JSON. Sessions were managed server-side with express-session.

Vercel Deployment

Vercel is optimized for frontend deployments with serverless functions. The vercel.json configured SPA rewrites so React Router handles all paths.

{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}

This catch-all rewrite is essential for any single-page app deployed to Vercel — without it, refreshing a route like /quiz returns a 404.

Architecture Decision: Why This Stack?

  • Vite — Fast builds, native ESM, instant HMR. Beats CRA and Webpack significantly in dev speed.
  • React + TypeScript — Component model fits trivia UI well. TypeScript catches data structure bugs early.
  • Tailwind CSS — Utility classes mean no context-switching between CSS files. Great for rapid prototyping.
  • SQLite — Zero-config database, good for MVP. Later replaced by Supabase for production.

Reflection

Week 1 set the foundation. The stack choices here (Vite + React + TypeScript + Tailwind) persisted through the entire project. The Express/SQLite backend, however, was temporary — it would be replaced in Week 3 with Supabase. This is a common pattern: start simple, validate the idea, then swap out infrastructure as needs grow.
