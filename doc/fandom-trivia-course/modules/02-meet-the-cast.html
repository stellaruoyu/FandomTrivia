Meet the Cast

The components that bring Fandom Trivia to life.

A Blockbuster in One Script

Here's a surprise: almost the entire app lives in a single file — src/App.tsx, about 2,300 lines. That's like an entire movie shot in one room. It's unusual for a production app, but it makes the code easy to explore.

Inside that file are several components — think of them as actors in a play. Each one handles a different "scene" in the app.

🧠 Why this matters: When you tell an AI coding tool "add a new view" or "change the quiz layout," knowing which component does what helps you give precise instructions. Instead of "change the quiz page," you can say "modify the MCQuizView component to add a timer."

The Main Characters

🏠

LandingView

The front door. Shows universe cards, hero section, and feature highlights. Users pick a fandom here.

📋

MCQuizView

The quiz master. Handles question display, answer selection, scoring, progress tracking, and saving results.

📊

DashboardView

"Pro Hub" — live leaderboard, active tournaments, stats feed. The competitive social layer.

🏅

RankingsView

Global leaderboard page. Pulls top scores from Supabase and displays them with gradient styling.

✨

ParticleCanvas

Background animations. Creates floating particles (ambient mode) and celebratory confetti (when you finish a quiz).

👤

Navbar + Modals

The top navigation bar, plus popups for quiz history, badges, username setup, and info screens.

How They Talk to Each Other

The components don't call each other directly. Instead, the React Router decides which one to show based on the URL. Think of it like TV channels — each route is a channel, and App.tsx is the remote control.

📱 Inside App.tsx — The Conversation
LandingView: "User just landed! Show them the universe cards."
Route: /
MCQuizView: "They picked Harry Potter. Loading questions from constants.ts..."
Route: /trivia-harry-potter
ParticleCanvas: "I'll set up ambient floating particles while they play."
Mode: ambient
MCQuizView: "Quiz complete! They got 8/10. Time to save the score."
Score: 80%
ParticleCanvas: "Celebration mode! Confetti time! 🎉"
Mode: celebration
DashboardView: "New score saved to Supabase. Updating leaderboard..."
Route: /dashboard
Navbar: "User checks their badges. Opening BadgesModal..."
Action: click

The "Remote Control" in Code

Here's the actual code from App.tsx that decides which component to show based on the URL.

CODE 

import { Routes, Route } from 'react-router-dom';
import ParticleCanvas from './ParticleCanvas';
import { supabase } from './supabaseClient';

const App = () => {
  return (
    <Routes>
      <Route path="/" element={<LandingView />} />
      <Route path="/trivia-harry-potter"
        element={<MCQuizView ... />} />
      <Route path="/rankings"
        element={<RankingsView />} />
      <Route path="/dashboard"
        element={<DashboardView />} />
    </Routes>
  );
};
PLAIN ENGLISH

Import the routing system — think of this as the TV channel switcher.

Import the particle animation component (it floats in the background).

Import the database connection (for login and saving scores).

The App component — the main container for everything.

The router container — only ONE route shows at a time.

When URL is "/", show the LandingView (homepage with universe cards).

When URL is "/trivia-harry-potter"...

...show the quiz view with Harry Potter questions.

When URL is "/rankings"...

...show the global leaderboard.

When URL is "/dashboard"...

...show the Pro Hub with tournaments and stats.

Close the router container.

End of the component's output.

End of the App function.
