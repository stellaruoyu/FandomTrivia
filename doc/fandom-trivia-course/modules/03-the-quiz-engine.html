How a Quiz Flows

Every quiz follows the same three phases. The app tracks where you are with React state — variables that remember what's happening right now.

1

Active

You see the question and options. Nothing selected yet.

state: selected = null

→
2

Answered

You pick an option. The app instantly shows correct/incorrect feedback.

state: selected = "Sunlight Sisters"

→
3

Finished

All questions answered. Results screen with score, grade, and character.

state: finished = true

🧠 Why this matters: This "state machine" pattern — idle → active → done — is everywhere in software. Wizards, checkout flows, multi-step forms — they all track where you are the same way. Once you recognize this pattern, you can debug any multi-step feature by asking: "What state is it in right now?"

What Happens When You Tap an Answer

Here's the actual code that runs the instant you select an answer option. This is from MCQuizView in App.tsx.

CODE 

const handleSelect = (option: string) => {
  if (selected) return;
  setSelected(option);
  const isCorrect = option.toLowerCase()
    === q.answer.toLowerCase();
  setScores(prev => ({
    ...prev,
    [currentQ]: isCorrect ? 'correct'
      : 'incorrect'
  }));
  if (isCorrect) {
    playCorrectSound();
  } else {
    playIncorrectSound();
  }
};

(keeping all previous scores intact via the spread operator)...

marking this one as 'correct' if you got it right...

or 'incorrect' if you got it wrong.

play the cheerful "da-ding!" sound.

Badges: The Achievement System

After each quiz, the app checks if you've earned any new badges. Here's the badge-checking logic — it runs right after the quiz complete screen appears.

CODE 

const evaluateBadges = (
  quizId: string, scorePct: number
) => {
  BADGES.forEach(badge => {
    if (unlockedBadgeIds
      .includes(badge.id)) return;

    if (badge.id === 'perfect_score'
      && scorePct === 100) {
      unlocked = true;
    }
    if (badge.targetQuiz) {
      const titleLower = quizId
        .toLowerCase();
      if (titleLower
        .includes(badge.targetQuiz)) {
        unlocked = true;
      }
    }
  });
};

This function runs after every quiz completion.

It receives the quiz name and your percentage score.

Look at every badge defined in the system.

Skip if you already unlocked this badge.

(no duplicate badges!)

Special case: "Perfectionist" badge.

Only unlock if you got 100%.

Mark as unlocked.

For universe-specific badges (e.g., "Forks Resident"):

Normalize the quiz name to lowercase...

(to handle case differences).

Does the quiz title contain

the badge's target fandom name?

If yes, you earn it!
