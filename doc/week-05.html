Constants Refactoring

As trivia content grew, constants.ts was reorganized into a maintainable structure:

// constants/
//   index.ts         — re-exports everything
//   universes.ts     — universe metadata + config
//   questions.ts     — all quiz questions by universe
//   badges.ts        — badge definitions
//   navigation.ts    — nav link config

This modular split meant each file stayed focused. Adding a new universe meant touching only universes.ts and questions.ts — no routing code, no component changes.

Development Rhythm

The commit pattern this week reveals a build → break → fix → refine cycle:

  1. Bold attempt — large commit adding new feature or restructuring
  2. Edge case discovery — something breaks in a new scenario
  3. Quick fix — targeted commit addressing the issue
  4. Polish pass — cleanup, rename, reorganize

This is normal for active development. The key insight: each iteration left the codebase slightly better than before, not degraded.
